The batch installed operators only. It did not create `FlowCollector`,
`FlowMetric`, `MonitoringStack`, `UIPlugin`, Perses,
`ObservabilityInstaller`, `MultiClusterObservability`, object storage,
retention config, or tenant workloads.

## Preflight

Preflight checked hub and spoke health, OADP, storage classes, PVC posture,
existing observability resources, package resolution, related image prefixes,
GitOps render, and server-side dry-run.

The install was approved because cluster health was steady and package
resolution was deterministic from the mirrored Red Hat catalog.

## Promotion Decision

Keep the operator-only install. It establishes the observability operator
foundation without activating flow collection, durable observability storage,
UI plugins, or tenant workloads.

Rollback is GitOps-first. Full CRD or console-plugin cleanup requires a
separate issue.
