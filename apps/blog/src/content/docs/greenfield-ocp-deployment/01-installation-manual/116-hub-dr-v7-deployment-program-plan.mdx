This chapter records the deployment program for `hub-dr-v7`, before live
deployment work begins.

The goal is a fully validated, documented, reproducible warm standby
management cluster for `hub-dc-v7`.

## Certification Boundary

In this workstream, certified means internally certified for the CompTech lab:

- every phase has evidence;
- every live change has an issue and rollback path;
- every important state is reproducible from recorded inputs;
- OpenShift health, GitOps health, operator health, backup readiness,
  compliance scans, security checks, and failover readiness are validated;
- documentation exists locally and in this public installation manual.

This is not an external Red Hat certification, PCI DSS ROC/AOC, QSA, ISO, or
bank-auditor attestation. External attestation requires a separate auditor-led
track.

## Non-negotiable Controls

- `hub-dr-v7` stays warm standby until a failover issue explicitly promotes it.
- Do not clone `hub-dc-v7` identity: no copied etcd, cluster ID, kubeconfigs,
  OAuth secrets, service-account signers, RHACS runtime secrets, or ACM
  runtime state.
- No live mutation without a phase issue, milestone, ADR coverage, validation
  plan, rollback path, and closeout.
- Do not print or commit secrets, kubeconfigs, pull secrets, PATs, Vault
  tokens, private keys, passwords, or full Secret manifests.
- Durable OpenShift state must be GitOps-first.
- Every material cluster gate must update this installation manual before
  closeout.

## Gate Model

Every phase must close with:

- issue comment with what changed;
- validation evidence;
- files touched;
- live systems touched;
- rollback or promotion decision;
- residual risks;
- next gate;
- local session record;
- updated local memory files;
- blog chapter or blog update when the gate is material.

## Final Done Criteria

`hub-dr-v7` is complete only when all are true:

1. cluster is installed and healthy;
2. platform GitOps is synced and healthy;
3. management services are installed in standby-safe mode;
4. backup, restore readiness, and secret custody are proven;
5. compliance and hardening evidence is closed or explicitly excepted;
6. operational readiness checks pass;
7. failover runbook dry-run is complete;
8. controlled takeover drill is completed or explicitly deferred;
9. public installation manual and local runbooks are complete;
10. reproducibility evidence pack is complete.
