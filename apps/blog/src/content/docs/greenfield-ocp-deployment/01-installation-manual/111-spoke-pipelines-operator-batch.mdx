## Promotion Decision

Keep the install. GitOps was synced, OLM was `AtLatestKnown`, the CSV was
`Succeeded`, `TektonConfig` was Ready, operator pods were Running, and spoke
cluster health stayed steady.

Rollback is GitOps-first:

1. Remove the Pipelines operator overlay from the spoke kustomization.
2. Remove `idms-openshift-pipelines` only if no other workload needs those
   mirror mappings.
3. Sync and validate OLM/operator pruning.
4. Do not force-delete Tekton CRDs or operator-created cluster-scoped objects
   without a separate cleanup issue.
