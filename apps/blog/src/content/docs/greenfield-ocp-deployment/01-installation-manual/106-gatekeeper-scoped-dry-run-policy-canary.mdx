This chapter records `OP-GF-OPERATORS-15`, the first persisted Gatekeeper
policy canary after the dry-run-only candidate design.

The canary deliberately stays non-enforcing. It persists one
`ConstraintTemplate`, one generated constraint kind, and one dry-run
constraint scoped only to a labeled namespace on `spoke-dc-v7`.

## Scope

The canary uses the candidate from chapter 105:

```text
K8sDisallowPrivilegedContainers
```

The policy flags Pods whose regular, init, or ephemeral containers set:

```yaml
securityContext:
  privileged: true
```

The constraint is dry-run only:

```yaml
spec:
  enforcementAction: dryrun
```

It only matches namespaces labeled:

```text
gatekeeper.comptech-lab.com/dryrun-candidate=true
```

The first labeled namespace is:

```text
gatekeeper-policy-canary
```

No hub policy object, deny enforcement, mutation config, broad namespace
selector, OADP object, node drain, or storage object was added.

## GitOps Rollout

The rollout was split into two functional commits so Gatekeeper could create
the generated constraint CRD before Argo CD applied the constraint instance.

Step 1 added the canary namespace, the `ConstraintTemplate`, and narrow Argo
CD RBAC:

```text
196fad3 Add spoke Gatekeeper dry-run template canary
```

After Step 1 reconciled, `spoke-dc-v7` had:

```text
namespace=gatekeeper-policy-canary label=true
constrainttemplate=k8sdisallowprivilegedcontainers
crd=k8sdisallowprivilegedcontainers.constraints.gatekeeper.sh established=True
constraint_count=0
gatekeeper-audit=1/1
gatekeeper-controller-manager=3/3
nonrunning_gatekeeper_pods=0
```

Step 2 added the dry-run constraint:

```text
4403e53 Add spoke Gatekeeper dry-run constraint canary
```

The platform changelog was updated separately:

```text
3a8017c Document spoke Gatekeeper dry-run canary
```

## Residual Risks

Gatekeeper policy enforcement is still not active. This gate creates audit
surface only.

The policy has only been introduced on `spoke-dc-v7`. The hub remains in the
no-policy Gatekeeper posture.

Gatekeeper still lacks durable disconnected image mirror coverage.
