## Decision Summary

The accepted design keeps the same logical structure as `spoke-dc-v7` while
allowing a smaller physical footprint:

- compact three-node VM-only topology;
- same GitOps and operator model where it applies;
- CNPG/database-layer replication for the insurance workload;
- ODF or local storage for PVCs on the DR cluster;
- no default ODF block-level replication for the database PVCs;
- active/passive traffic control with split-brain prevention;
- distinct cluster identity and separate secret custody.

The active workload remains `spoke-dc-v7` until a later controlled failover
drill promotes `spoke-dr-v7`.

## Non-Goals

- Do not create VMs, DNS records, HAProxy entries, install artifacts, GitOps
  overlays, ACM imports, or live OpenShift resources in this gate.
- Do not clone the active cluster identity or secret material.
- Do not route traffic to `spoke-dr-v7` yet.
